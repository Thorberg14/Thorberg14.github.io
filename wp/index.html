Task 2 - Transformation of chosen document to DocBook format

Look at the result!

Document, which I have chosen for transformatinon, is my Bachelor thesis documentation. This document was made using Texmaker, which uses LaTeX format. So, it should not be a problem to convert this format to DocBook.

I used XMLmind as editor for creating it. I already had book template consisting of title page, one chapter, appendix and bibliography. This I used as base ground for further work. Most of the work consist just from copying text from LaTeX document and rewriting LaTeX sytax to DocBoox syntax. For example, in Latex I used \emph{text} or \textbf{text} for emphasizing the important word. In DocBook it is nearly the same - <emphasis>some text</emphasis>. For bold text I had to use <emphasis role="bold"> tag. Unfortunetly, I did not find simple tag for line break as \newline or \\ in LaTeX. Therefore I used <literallayout> tag for inserting texts, in which I need to have control over the line breaks. Citation and references construction was also very similar to LaTeX.

The main structure of the DocBook document consits of Chapters, which can be then splitted into sections <Section> tag. Sections consist mostly of sub-sections ( <section> tag again) paragraphs (<para> tag), figures ( <figure> ) and item lists ( <itemizedlist mark="opencircle"> ).

Additional features, which I did not used in LaTeX was Book Index, in which the selected words can be added. Then it shows sorted alphabetical list of the word and the page where the word is used. For this feature is needed <index/> to show the list, and <indexterm><primal>Word</primal> <secondary>optional secondary</secondary> <indexterm> construction to add the word to the list. It is important, where this cosntruction is written. The shown page number depends on it.

Task 3 - XML presentation

My task was to design DTD for XML presentation, then create example of presentation and finally transform it via XSL to HTML and PDF format.

DTD which I created consits of basic elements for structure, <slide>, <para> for paragraph and <list> for list of items. Naturally, I added few elements for text style, like bold and italic text. In element <meta> must be added presentation title and author name. there can be also added company name and logo, and date.

Slide must have one header. After that can consits of images, lists, paragraphs, and empty lines

For adding image, there is needed to add url. Image can have also defined size and caption

For whole presentation, there can be defined sources which was used in presentation. It starts with tag <sources>. If it was used, in the end of final presentation will be added one more slides with the list of sources.
